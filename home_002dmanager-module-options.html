3.4 services.darkman.activationScript.dark

Commands to run when dark mode is activated.

The scripts are placed in a Nix derivation, dark-mode.d/nix-hm-module-darkman.sh, which is then added to xdg.systemDirs.data ($XDG_DATA_DIRS). The interpreter of these script is pkgs.bash, as they are created by pkgs.writeShellScript.

Type: string

Default: ""

Declared by: - modules/hm/darkman.nix

3.10 services.darkman.config.useDbus

Whether to expose the current mode via darkman’s own D-Bus API. The command line tool uses this API to apply changes, so it will not work if this setting is disabled.

You will need to add pkgs.darkman to your NixOS services.dbus.packages for this to work.

Type: null or boolean

Default: true

Declared by: - modules/hm/darkman.nix

Previous: services.darkman.config.useDbus, Up: home-manager module options  

3.11 services.darkman.config.useGeoclue

Whether to use a local geoclue instance to determine the current location.

You must also enable the system-wide geoclue2 service on NixOS: services.geoclue.enable = true;

Type: null or boolean

Default: false
